# 球队专区功能实现计划

## 需求分析

用户要求：

1. 把马刺专区放到球队列表下面
2. 以马刺专区为模板建立全部三支球队（马刺、湖人、火箭）的专区

## 现有资源分析

### 当前马刺专区结构 (spurs.html)

* 球队阵容 (roster)

* 赛季赛程 (schedule)

* 赛季统计 (stats)

* 球星专区 (文班亚马)

### 可用CSV数据文件

* `1989-2026SAS.csv` - 马刺历史数据

* `1989-2026LAL.csv` - 湖人历史数据

* `2023-2024spurs.csv`, `2024-2025spurs.csv`, `2025-2026spurs.csv` - 马刺赛季数据

## 实现计划

### 任务1: 修改首页index.html

* 在球队列表下方添加三支球队的专区入口

* 设计统一的专区卡片样式

* 添加导航链接到各球队专区

### 任务2: 创建湖人专区页面 (lakers.html)

* 复制马刺专区模板

* 修改主题颜色为湖人配色（紫色/金色）

* 更新队徽和队名显示

* 添加科比·布莱恩特专区

* 配置API调用路径

### 任务3: 创建火箭专区页面 (rockets.html)

* 复制马刺专区模板

* 修改主题颜色为火箭配色（红色/白色）

* 更新队徽和队名显示

* 添加姚明/哈登专区

* 配置API调用路径

### 任务4: 更新马刺专区页面 (spurs.html)

* 确保与其他两支球队风格一致

* 优化文班亚马专区内容

### 任务5: 扩展后端API支持多球队

* 修改或创建多球队数据API

* 添加湖人CSV数据读取支持

* 添加火箭CSV数据读取支持

## 设计规范

### 配色方案

* **马刺**: 银/黑 (#C4CED4 / #000000)

* **湖人**: 紫/金 (#552583 / #FDB927)

* **火箭**: 红/白 (#CE1141 / #FFFFFF)

### 页面结构

```
球队专区卡片
├── 球队Logo
├── 球队名称
├── 核心球星展示
└── 快速链接按钮
```

## 风险处理

1. **数据文件缺失**: 检查CSV文件是否存在，如不存在使用模拟数据
2. **API兼容性**: 确保新API与现有前端调用兼容
3. **样式冲突**: 使用独立的CSS命名空间避免样式冲突

## 预期结果

完成后首页将显示：

1. 全联盟球队列表（保留原有功能）
2. 三支重点球队的专区入口（马刺、湖人、火箭）
3. 每支球队专区包含完整的阵容、赛程、统计和球星内容

统一计量标准，讲英制数据转换成公制数据，主要存在于身高数据，臂展数据
